Traffic Sign Assistance KPI Evaluation

This project focuses on evaluating the performance of a Traffic Sign Assistance (TSA) system by comparing predicted speed limits from sensor data with ground truth labels over mapped driving routes.

Data Pipeline & Integration

  • Use bag file tags for detected speed limits and GPS coordinates
  • Integrate KML files recorded from GPS logs
  • Query ground truth speed limits from a labeling API
  • Reference OpenStreetMap (OSM) to align roads and regions

KPI Computation

For every segment along the route:

  • Compare predicted vs. true speed limits over time
  • Compute distance traveled using longitude/latitude or the KML trace
  • Mark segments as true positive or false detection
  • Calculate KPI using: (Correct Distance / Total Distance) × 100%

Dashboard & Output

The results are visualized in a TSA KPI dashboard showing:

  • Accuracy per trip
  • False detection zones
  • Route overlays with OSM context

Preview



This KPI evaluation tool plays a key role in validating the reliability of TSA systems in real-world driving conditions, ensuring compliance with speed regulations and improving road safety.
